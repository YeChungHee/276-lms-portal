# 어드민 — 퀴즈 점수·채점 상세 + 학습 분석 구성 계획서 (v3 · Enhanced)

> **버전:** v3.0 — 기존 admin.html은 **참조(baseline)** 로만, 그 이상의 기능을 추가
> **기준일:** 2026-05-30
> **적용:** 전 퀴즈 6종 공통 + 신규 어드민(admin-scores.html) 통합
> **연계:** `01_Admin_Portal_Plan.md`, `Code.gs`(v3.0)

---

## 0. 방향

기존 `admin.html`의 "응시자 목록 → 상세 보기(문항·응답·정답·결과·해설) → Q별 오답률 → 재채점"은 **검증된 기본기**다. 이를 골격으로 삼되, **단순 채점 열람을 넘어 '학습 분석·개인화·실행(보강)'까지 잇는 한 단계 위 기능**을 더한다.

| 구분 | 기존(baseline) | v3 강화 |
|---|---|---|
| 점수 | 정답수·점수·등급 | + 통계(평균/중앙값/표준편차)·백분위·재응시 추이 |
| 문항 | 응답·정답·결과·해설 | + **진단태그 칩 · 관련 교재 딥링크 · 문항 변별도/난이도** |
| 오답률 | Q별 오답률 그리드 | + **태그별 취약 히트맵 · 문항 신뢰도** |
| 실행 | (없음) | + **오답노트 자동 생성 · 보강과제 연결 · 재시험 요청(Slack) · 미응시자 추적** |
| 범위 | 핀테크 OX 단일 | **전 퀴즈 멀티 + 사원 360 카드** |

## 1. 참조한 기존 구성 (그대로 계승)

- 응시자 목록 표(# · 성명 · 응시일시 · 정답수 · 점수 · 등급 · 상세) + 이름 검색 + CSV
- 상세 모달: 요약 카드 + 문항별 표(문항·응시답안·정답·결과·해설) + **오답행 강조**
- **재채점 원칙**: 시트 저장값이 아닌 정답 기준 재산출(정답 수정 시 자동 반영)
- Q별 오답률 그리드(히트맵)

---

## 2. 더 나은 기능 (v3 핵심 차별화)

### 2-1. 문항 상세 모달 — "복습이 되는" 상세
문항별 행에 다음을 추가:
- **진단 태그 칩**(#전자어음 등) — 어느 영역에서 틀렸는지 즉시 인지
- **"관련 교재 보기" 딥링크** — 해당 태그/문항이 다루는 **교과서 챕터로 바로 이동**(예: #전자어음 → textbook-enote-discount.html#ch3)
- **응답 분포**(선택): 같은 문항을 푼 전체 사원 중 각 보기 선택 비율(객관식) — "다들 헷갈리는 오답"
- 오답만 모아보기 토글 · 모달 내 **인쇄/PDF**

### 2-2. 문항 분석(Item Analysis) — 출제 품질 진단
- **문항별 정답률** + 난이도 자동 라벨(쉬움 ≥80% / 보통 / 어려움 ≤40%)
- **변별도**: 상위 27% vs 하위 27% 그룹의 정답률 차 → 변별력 낮은/역변별 문항 플래그(출제 개선 신호)
- **신뢰도 지표**(가능 시 KR-20 근사) — 퀴즈 자체 품질 점검

### 2-3. 취약점 히트맵 — 조직 + 개인
- **사원 × 진단태그 매트릭스**(태그별 정답률, 색상 히트맵)
- 조직 공통 취약 태그 Top N → **교육과정 보완 인사이트**
- 개인 취약 태그 → 2-5 보강과제로 자동 연결

### 2-4. 사원 360 카드 — 한 사람의 전체 학습
한 사원 클릭 시: **모든 퀴즈 점수 + 교육완료 현황 + 취약 태그 + 재응시 추이**를 한 화면에. (퀴즈·교과서 데이터 통합)

### 2-5. 실행(Action) 자동화
- **오답노트 자동 생성**: 사원별 틀린 문항(문항·정답·해설·태그)만 모은 노트 → 인쇄/Slack 발송
- **보강 과제 자동 매칭**: 취약 태그 → 해당 교과서 챕터 + 추가 문항을 묶어 `remediation_tasks` 생성(기존 add_remediation 활용)
- **합격선 관리**: 퀴즈별 합격 기준(예 80%) 미달 사원 자동 플래그 + **재시험 요청**(Slack DM stub)
- **미응시자 추적**: `employees` ↔ 응시 기록 대조 → "아직 안 본 사원" 목록

### 2-6. 통계·내보내기 강화
- 통계: 평균·중앙값·표준편차·합격률·백분위(개인 위치)
- 내보내기: 응시 요약 CSV + **문항별 상세 CSV** + 오답노트(PDF/인쇄)
- 필터: 퀴즈별·기간별·등급별·합격여부 / 정렬·검색

## 3. 데이터 모델 (확장)

응시 1건 = 1행 + **`상세JSON`**(문항 배열) + 선택 메타.
```json
{ "n":1, "q":"질문", "type":"ox|mc", "sel":"응답", "ans":"정답",
  "ok":true, "tag":"진단태그", "exp":"해설" }
```
- 응시 레벨 추가 필드(선택): `소요초`(문항/총), `시도회차`(재응시 추이용)
- 레거시 핀테크 OX는 시트 Q1~Q30 + 어드민 내장 `CORRECT/QUESTIONS/EXPLANATIONS`로 동일 형식 재구성(딥링크 태그 매핑 포함)

## 4. 프론트(퀴즈 6종) 변경

`grade()`에서 문항별 detail 수집(+선택: 문항별 소요시간) → `quiz_submit` 전송. 6종 `QUIZ[]` 구조 동일 → 공통 패치.
```js
const detail = QUIZ.map((it,i)=>({n:i+1,q:it.q,type:it.type,
  sel:selText(i),ans:ansText(it),ok:isOk(i),tag:it.tag,exp:it.exp}));
// quiz_submit: {action,name,quiz,correct,pct,grade,weakTags,detail,(elapsed)}
```

## 5. 백엔드(Code.gs) 변경

- `quiz_submit`: `상세JSON`(+소요시간·회차) 저장, 헤더 확장
- `get_attempt(name,ts,quiz)`: 단건 상세
- `get_item_stats(quiz)`: 문항별 정답률·변별도(서버 집계) — 또는 어드민에서 detail 집계
- `get_scores`: 목록(요약, 레거시/신규 호환 유지)
- 레거시 OX: Q1~Q30 반환 → 어드민이 CORRECT/EXP로 문항 분석까지 산출

## 6. 어드민 화면 구성 (admin-scores.html 통합)

상단 **퀴즈 선택 + 기간/등급 필터** → 아래 4블록:
1. **통계 바**: 응시자·평균·중앙값·표준편차·합격률·최고
2. **문항 분석**: Q별 정답률·난이도·변별도(플래그) — 히트맵
3. **취약 태그 히트맵**: 사원×태그 + 조직 Top 취약
4. **응시자 목록**(검색·정렬·CSV) → 행 **[상세 보기]** → **강화 모달**(요약 + 문항표 + 태그칩 + 교재 딥링크 + 오답노트/인쇄)
5. **사원 360**(성명 클릭) · **미응시자** · **보강/재시험 액션**

디자인: 보라 테마 · 맑은 고딕 · **SVG 아이콘(이모지 금지)**.
