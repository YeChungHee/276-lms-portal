## 2. 어드민 개요 및 목표

어드민 포털은 분산된 관리 기능(콘텐츠 제작, 계정 승인, 진도·점수 모니터링, 취약점·보강)을 **단일 운영 콘솔**로 통합한다. 현재 세 개의 관리 화면이 개별 존재하므로, 이를 공통 IA·디자인으로 묶는 것이 1차 목표다.

| 목표 | 설명 |
|---|---|
| 통합 콘솔 | 계정·콘텐츠·진도·점수·취약점을 한 진입점에서 운영 |
| 콘텐츠 자가 관리 | 관리자가 교과서·퀴즈를 직접 등록·배포 |
| 계정 기반 추적 | 사원 계정별 진도·교육완료·퀴즈 점수·취약 태그 집계 |
| 운영 자동화 | 계정 승인→임시PW→Slack 전달, 보강 과제 자동 생성 |
| 권한 보호 | 관리자 인증(현행 `ADMIN_KEY`) → 추후 관리자 계정 로그인 격상 |

## 3. 어드민 정보구조(IA) — 메뉴 구성

상단 공통 topbar + 좌측/탭 네비게이션. 7개 메뉴로 구성한다.

| # | 메뉴 | 현행 구현 파일 | 상태 |
|---|---|---|---|
| 1 | **운영 대시보드**(요약) | (신규) `admin.html` 홈 | 계획 |
| 2 | **계정 관리**(요청 승인·생성) | (신규) | 계획 |
| 3 | **교과서 관리** | `admin-content.html` (탭1) | ✅ 구현 |
| 4 | **질문지(퀴즈) 관리** | `admin-content.html` (탭2) | ✅ 구현 |
| 5 | **교육 진도·완료 현황** | `admin-progress.html` | ✅ 구현 |
| 6 | **퀴즈 점수·취약점 분석** | (부분: 기존 `admin.html`) | 부분 |
| 7 | **운영**(공지·Slack 로그·내보내기) | (신규) | 계획 |

> 통합 방향: 위 메뉴를 하나의 `admin.html` 셸(좌측 네비)에서 탭/iframe 또는 페이지 링크로 묶고, 공통 topbar로 상호 이동. 현재는 각 페이지 topbar 링크로 연결되어 있음(교과서·질문지 ↔ 교육완료 현황 ↔ 사원 포털).

## 4. 메뉴별 상세 기능

### 4-1. 운영 대시보드(요약)
관리자 진입 시 첫 화면. 핵심 KPI 카드 + 알림.

- KPI: 활성 사원 수, 평균 교육완료율, 평균 퀴즈 점수, 보강 필요 인원, 계정 승인 대기 건수
- 최근 활동: 최근 교육완료/퀴즈 응시/계정 요청 타임라인
- 바로가기: 승인 대기, 취약 사원 Top, 미배포 콘텐츠

### 4-2. 계정 관리
- **요청 목록**: `account_requests` 조회(대기/승인/반려)
- **계정 생성**: 중복 검증 → 로그인ID·이메일ID 확정 → **임시 비밀번호 발급**(`Tmp#XXXX`) → `employees` 등록
- **Slack/이메일 전달**: 로그인 URL·아이디·임시PW 발송
- **계정 상태**: 활성/비활성/비밀번호 초기화
- 엔드포인트: `admin_requests`(GET) · `create_account`(POST) · `signup`(사원측)

### 4-3. 교과서 관리 *(구현됨 — admin-content.html 탭1)*
- 등록 방식 토글: **웹페이지 링크 | PDF 업로드**
- 웹링크: 제목·URL·주차·챕터 수·열기 방식·부서 → 등록
- PDF: 업로드 → 텍스트 추출(pdf.js) → 편집 → 배포(초안)
- 목록: 교재 7종 사전 등록, 활성/비활성·삭제·열기
- 엔드포인트: `save_textbook` · `get_textbooks`
- **대시보드 연동 완료**: `admin_textbooks` → 사원 대시보드 교과서 목록 반영

### 4-4. 질문지(퀴즈) 관리 *(구현됨 — admin-content.html 탭2)*
- 퀴즈 생성(퀴즈명·연결 교재·합격 기준) + 문항 추가(질문·유형 OX/객관식·보기·정답·**진단 태그**·난이도·해설)
- 문항 목록·태그 필터·활성/비활성·배포
- 등록 퀴즈 7종(핀테크OX30·플랫폼15·프로세스15·심사채권추심20·경제10·전자어음10 등)
- 엔드포인트: `save_quiz` · `add_questions` · `deploy_quiz` · `get_questions`

### 4-5. 교육 진도·완료 현황 *(구현됨 — admin-progress.html)*
- **사원 × 교과서 매트릭스**: 셀 상태 = 교육완료 / n단계 진행 / 미시작
- 컬럼: 회사소개·경제·금융발달·핀테크·매출채권·플로우페이 플랫폼·플로우페이 프로세스·심사·채권추심
- 요약 카드: 총 사원·평균 완료 교과서·전체 완료율
- 이름 검색·CSV 내보내기
- 데이터 출처: `get_completions`(GET) — 진도명 접두사 `[교과서명] …` / `✓ 교육완료` 파싱

### 4-6. 퀴즈 점수·취약점 분석·보강 교육
- 사원별 퀴즈 점수·등급·응시 이력(`quiz_results`)
- **진단 태그별 정답률** 집계 → 70% 미만 취약 영역 식별
- **보강 교육 자동 생성**: 취약 태그 → `remediation_tasks`(보강 제목·자료 링크·추가 문항·완료 기준)
- 현행: 기존 `admin.html`이 OX 퀴즈 응시·등급 분포를 표시(부분 구현). 신규 자가채점 퀴즈 점수의 서버 저장(`quiz_submit`) 연동이 필요.
- 엔드포인트: `quiz_submit`(사원측 저장) · `admin_overview`/통계(GET)

### 4-7. 운영
- 공지 등록(대시보드 상단 노출), 모듈 노출 제어
- **Slack 발송 이력**(`slack_logs`) 조회
- 데이터 내보내기(CSV), 교육과정(커리큘럼)·주차 편성

## 5. 데이터·백엔드 연동 (Code.gs v3.0)

이미 구현된 엔드포인트(18종)와 어드민 메뉴 매핑.

| 메뉴 | 사용 엔드포인트 | 시트 |
|---|---|---|
| 계정 관리 | `signup`·`admin_requests`·`create_account`·`change_password` | account_requests, employees |
| 교과서 관리 | `save_textbook`·`get_textbooks` | textbooks |
| 질문지 관리 | `save_quiz`·`add_questions`·`deploy_quiz`·`get_questions` | quizzes, quiz_questions |
| 진도·완료 | `progress`(사원)·`get_progress`·`get_completions` | 학습진도 |

> 배포 전제: Apps Script `initAllSheets()` 1회 실행 → 웹앱 배포(모든 사용자) → 각 화면 `APPS_SCRIPT_URL` 반영. 인증은 현행 `ADMIN_KEY=flowpay2026`.

## 7. 권한·화면 흐름

```
admin.html(관리자 인증)
 ├─ 운영 대시보드(요약)
 ├─ 계정 관리 ───────── create_account → Slack 전달
 ├─ 교과서 관리(admin-content #book)
 ├─ 질문지 관리(admin-content #quiz)
 ├─ 진도·완료 현황(admin-progress)
 ├─ 퀴즈 점수·취약점 → 보강 과제 생성
 └─ 운영(공지·Slack 로그·내보내기)
```

- 인증: 진입 시 ADMIN_KEY 확인(추후 관리자 계정). 민감 액션(계정 생성·배포)은 키 재검증.
- 권한 등급(2차): 슈퍼관리자 / 부서 리드(읽기·승인) 분리 검토.
